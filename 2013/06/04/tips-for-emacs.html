Some Tips for Emacs

June 4th, 2013

This file includes some tips for configuring emacs.

1 How to use Auto-complete to show the completion result of semantics.

Semantic is a powerful tool to do C/C++ parsing. But it uses the right-click menu to show completion which I think is not beautiful. Auto-complete’s menu is much more elegant, so can we use Auto-complete’s menu to show Semantic’s result? The answer is yes. Auto-complete’s author already thought about this and there is a function called ac-complete-semantic will do this for us automatically. Just do the following: Install both of the two add (define-key ac-mode-map (kbd “M-l”) ‘ac-complete-semantic) to a .emacs. Then everything works.

2 Use iswitchb to switch buffers.

I suggest to open iswitchb-mode all the time. Just insert

 
  (iswitchb-mode)

into your .emacs and then you will see this is much much better than the default one.

3 Use doxymacs to insert doxygen-like comments

First, install doxymacs (e.g. in Ubuntu):

 
   sudo apt-get install doxymacs

Add

 
  (require 'doxymacs)
  (add-hook 'c-mode-common-hook 'doxymacs-mode)

to the .emacs file. Then we can insert doxygen-like comments by keyboard shotcuts.

5 Semantic settings

I used to use semantics to complete c/c++ code in emacs but I found it does not work in a lot of cases. Recently, I found another extensions based on clang and llvm which performs very well (I only tried this one). It works perfectly as long as your code is correct in syntax. To install it, first you have to install clang. If you are using ubuntu, just using the command: sudo apt-get install clang. Apt can automatically select relevant llvm libs to install. Then just follow the readme files of the extension to config the .emacs file. Until now, the only drawback I found of this tool is that: compared with semantics/cedet, it does not include code navigation functions. So you can still use semantics for the code jumping functions.

This section just records some Semantic setting. Set the search directory, Reference

 
(defconst cedet-user-include-dirs
  (list ".." "../include" "../inc" "../common" "../public"
        "../.." "../../include" "../../inc" "../../common" "../../public" "/data0/local/include/opencv/"))
(defconst cedet-win32-include-dirs
  (list "C:/MinGW/include"
        "C:/MinGW/include/c++/3.4.5"
        "C:/MinGW/include/c++/3.4.5/mingw32"
        "C:/MinGW/include/c++/3.4.5/backward"
        "C:/MinGW/lib/gcc/mingw32/3.4.5/include"
        "C:/Program Files/Microsoft Visual Studio/VC98/MFC/Include"))
(require 'semantic-c nil 'noerror)
(let ((include-dirs cedet-user-include-dirs))
  (when (eq system-type 'windows-nt)
    (setq include-dirs (append include-dirs cedet-win32-include-dirs)))
  (mapc (lambda (dir)
          (semantic-add-system-include dir 'c++-mode)
          (semantic-add-system-include dir 'c-mode))
        include-dirs))

9 Multi-shell

I think the “shell” is the better than “multi-term”, “eshell”. But by default, one can only have one “shell”. The reason is that the buffer’s name is unique (always “shell”). If we change the shell’s name everytime after we created it. Then we can get multiple shells:

 
(setq shellnumber 0)
(defun newshell () (interactive) (progn 
            (shell)
            (rename-buffer (concat (number-to-string shellnumber) "shell"))
            (setq shellnumber (+ shellnumber 1))))
(global-set-key "\C-x." 'newshell)

10 “Auto-kill” *Completions* buffers

The *Completions* buffers can be annoying sometime, using following code can “auto-kill” them after openning a file or after commiting a command in shell:

 
; for file openning
(add-hook 'minibuffer-exit-hook 
      '(lambda ()
         (let ((buffer "*Completions*"))
           (and (get-buffer buffer)
            (kill-buffer buffer)))))
; for shell completion
(defun delete-completion-window-buffer (&optional output)                                                                
  (interactive)                                                                                                
  (dolist (win (window-list))                                                                                  
    (when (string= (buffer-name (window-buffer win)) "*Completions*")                                          
      (delete-window win)                                                                                      
      (kill-buffer "*Completions*")))                                                                          
  output)                                                                                                      
(add-hook 'comint-preoutput-filter-functions 'delete-completion-window-buffer)

11 Jump to previous window

I use C-x p to jump to previous window:

 
  (global-set-key (kbd "C-x p") '(lambda () (interactive) (other-window -1)))
